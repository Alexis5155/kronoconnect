# Changelog 0.0.2

## [0.0.2] - 2026-06-17

### Fixed
- Fixed layout overlapping issues on the authentication templates (login, register, forgot-password, reset-password, etc.) where the footer (and GDPR policy links) would render on top of the central card on smaller viewports. They now display beneath the card in the natural document flow and scroll accordingly.
- Fixed a 500 Server Error (`Undefined array key "role"`) when viewing a user's details page in the admin section (`/admin/users/{id}`) by using `UserModel::findById()` instead of a raw SQL query on the `users` table, ensuring the joined role/group information is fetched.
- Fixed an issue in `SsoController` where a super-administrator might not bypass app access controls correctly due to checking `Session::get('user')['role']` instead of the standard `Session::get('user_role')` session key.
- Fixed an issue where logging in or registering a WebAuthn security key triggered an "Undefined array key 'credential_id'" error by including `credential_id` in the `getWebAuthnCredentials` database query.
- Fixed the update system process where a missing CSRF token in the request body caused a "CSRF token invalide" error when applying updates. Also enhanced error handling in the update UI to stop execution and display errors instead of showing a false success message.
- Fixed a database migration system error where MySQL DDL statements implicitly committed the active transaction, causing subsequent `$db->commit()` calls to fail with a "There is no active transaction" exception.
- Fixed the database migration system parser to correctly support `DELIMITER` and multi-line statements in SQL migrations (such as stored procedures) by using a line-by-line SQL statement splitter.
- Improved the SQL database migration query parser with an ultra-robust state-machine to correctly parse multi-line quoted strings and comments containing semicolons.
- Fixed database migration `002_1.0.1.sql` to check both the unprefixed `kronoconnect_files` and prefixed `{PREFIX}kronoconnect_files` tables when renaming to `{PREFIX}files`, and dropping the empty newly created `{PREFIX}files` table if it was created in a previous failed run to prevent "Table already exists" errors.
- Fixed duplicate system update success notifications being sent to users belonging to multiple groups.
- Fixed an issue on the settings page where database migration metrics (applied, pending, total migrations) were calculated incorrectly by scanning an empty `delta/` folder instead of querying all registered migrations from `Migration::status()`.

### Added
- Added an inset in the updates settings page to retrieve and display the changelog of the last update.
- Replaced the browser native confirm popup with KronoConnect's custom styled confirmation modal when launching an update.
- Added automatic update completion notifications sent to all users with `kc.settings.update` permission (or `super_admin` role) upon successful updates.

### Changed
- Changed the SMTP settings card icon from a heart-themed envelope icon (`bi-envelope-paper-heart`) to a neutral envelope-at icon (`bi-envelope-at`).
- Changed the user profile chip in the topbar of the administration layout to link to the user profile page (`/profile`).
- Overhauled the system update visual interface: the installation process, including a progress bar and terminal logs, is now displayed directly within the update modal, replacing the legacy bottom-of-page terminal.
- Moved the updates tab ("Mises à jour") to the bottom of the navigation sidebar on the settings page.
