# Changelog 0.0.2

## [0.0.2] - 2026-06-17

### Fixed
- Fixed layout overlapping issues on the authentication templates (login, register, forgot-password, reset-password, etc.) where the footer (and GDPR policy links) would render on top of the central card on smaller viewports. They now display beneath the card in the natural document flow and scroll accordingly.
- Fixed a 500 Server Error (`Undefined array key "role"`) when viewing a user's details page in the admin section (`/admin/users/{id}`) by using `UserModel::findById()` instead of a raw SQL query on the `users` table, ensuring the joined role/group information is fetched.
- Fixed an issue in `SsoController` where a super-administrator might not bypass app access controls correctly due to checking `Session::get('user')['role']` instead of the standard `Session::get('user_role')` session key.
- Fixed an issue where logging in or registering a WebAuthn security key triggered an "Undefined array key 'credential_id'" error by including `credential_id` in the `getWebAuthnCredentials` database query.
