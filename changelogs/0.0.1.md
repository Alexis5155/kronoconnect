# Changelog 0.0.1

## [0.0.1] - 2026-06-14

### Fixed
- Fixed a fatal 500 error on case-sensitive file systems (production) by renaming the autoloader file `AutoLoader.php` to `Autoloader.php` (matching its PSR-4 class name `Autoloader`).
- Updated the require/use statements in `cron/gdpr_purge.php` and instantiated the Autoloader object correctly instead of calling `register()` statically.
- Relocated the installation check in `index.php` to the very top, before configuration imports. This avoids a 500 Internal Server Error when starting a fresh installation since configuration files (`app.php`) do not exist yet.
- Integrated global error and exception handling in `index.php` (similar to KronoCore) to catch and format unhandled exceptions (like database connection issues) instead of raising raw 500 errors.
- Improved the installer to dynamically detect the application's base URL instead of suggesting a hardcoded `/kronoconnect` path.
- Fixed a duplicate `/login` redirection issue in the installer's post-installation page.
- Renamed the `{PREFIX}kronoconnect_files` database table to `{PREFIX}files` to adhere to nomenclature standard (updated migrations, schema, FileManager, and AuthController).
- Updated the installer to generate a default `app.php` config file if it does not exist (as it is gitignored and absent on a fresh production clone), instead of returning silently.
- Fixed a broken layout/CSS issue on the login page by removing the hardcoded IP address in the `base_url` configuration property (`app/config/app.php`), and fixing a Windows backslash issue in URL auto-detection (`get_base_url`), enabling proper local URL detection.
- Fixed the installation wizard `/install` to correctly save SMTP/email configuration settings in the database.
- Fixed the installer SMTP test function to transmit all credentials and send a real test email if a destination address is specified.
- Fixed a fatal error during WebAuthn security key registration/deletion in `ProfileController.php` by importing the `Logger` class, preventing JSON parsing errors (unexpected token `<` due to HTML 500 error pages) by wrapping API responses in proper `Throwable` catch blocks.

### Added
- Integrated a new custom blue shield favicon (`favicon.png`) across all application layouts (`admin`, `auth`, `main`).
